# Git

# Commit 规范

# Header ⭐️

<type>:<subject>
1

Header 书写规范

type 用于说明 commit 的类别，可以使用如下类别：

  • feat：新功能（feature）
  • fix：修补 bug
  • doc：文档（documentation）
  • style： 格式（不影响代码运行的变动）
  • refactor：重构（即不是新增功能，也不是修改 bug 的代码变动）
  • test：增加测试
  • chore：构建过程或辅助工具的变动

subject 是 commit 目的的简短描述。

  • 以动词开头，使用第一人称现在时，比如 change，而不是 changed 或 changes
  • 第一个字母小写
  • 结尾不加句号（。）
